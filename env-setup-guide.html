Настройка файла .env для подключения к базе данных

📌 Что мы делаем на этом шаге?
Создаем файл .env с настройками подключения к PostgreSQL и вводим туда пароль от базы данных.
1
Копирование файла .env.example → .env

У нас есть файл-шаблон .env.example, который нужно скопировать в .env

🪟 Windows
🍎 macOS
🐧 Linux

Способ 1: PowerShell или CMD

copy .env.example .env

Способ 2: Вручную через Проводник

  1. Откройте папку проекта в Проводнике
  2. Включите показ скрытых файлов: Вид → Показать → Скрытые элементы
  3. Найдите файл .env.example
  4. Скопируйте его (Ctrl+C)
  5. Вставьте (Ctrl+V) - появится копия
  6. Переименуйте в .env

Через Terminal:

cp .env.example .env

Через Finder:

  1. Откройте папку проекта
  2. Нажмите Cmd + Shift + . (показать скрытые файлы)
  3. Скопируйте .env.example
  4. Переименуйте копию в .env
cp .env.example .env
✅ После этого шага у вас будет два файла:
• .env.example - шаблон (не трогаем)
• .env - ваш файл (будем редактировать)
2
Что находится в файле .env?

Давайте разберем каждый параметр:

Параметр Описание Менять?
DB_HOST Адрес сервера БД
Значение: localhost		 Не менять
DB_PORT Порт PostgreSQL
Значение: 5432		 Не менять
DB_NAME Имя базы данных
Значение: stolevel		 Не менять
DB_USER Пользователь PostgreSQL
Значение: postgres		 Обычно нет
DB_PASSWORD Пароль от PostgreSQL
⚠️ Тот, что вы создали при установке!		 ОБЯЗАТЕЛЬНО!
PORT Порт веб-сервера
Значение: 3000		 Если свободен
JWT_SECRET Секретный ключ для токенов
Любая длинная строка		 Для теста - нет
NODE_ENV Режим работы
Значение: development		 Не менять
3
Редактирование файла .env

Как выглядит файл ДО изменения:

1
# Database Configuration
2
DB_HOST=localhost
3
DB_PORT=5432
4
DB_NAME=stolevel
5
DB_USER=postgres
6
DB_PASSWORD=your_password ← ИЗМЕНИТЕ ЭТО!
7
8
# Server Configuration
9
PORT=3000
⚠️ ВАЖНО:
Замените your_password на пароль, который вы создали при установке PostgreSQL!

Как выглядит файл ПОСЛЕ изменения:

6
DB_PASSWORD=MySecretPass123 ← Ваш реальный пароль

5
Частые ошибки

❌ Неправильно:

DB_PASSWORD = MyPass123 ← Пробелы вокруг =
DB_PASSWORD=MyPass123 ← Пробел в начале
DB_PASSWORD="MyPass123" ← Кавычки (обычно не нужны)

✅ Правильно:

DB_PASSWORD=MyPass123
6
Проверка настроек

После сохранения файла .env, запустите сервер:

npm start

✅ Если все правильно:

✅ Подключено к базе данных PostgreSQL
🚗 СТО Level API Server
🌐 Сервер запущен на порту 3000

❌ Если есть ошибка:

❌ password authentication failed for user "postgres"
Решение: Проверьте пароль в .env
🎉 Готово! Теперь переходите к Шагу 7: Запуск сервера
npm install && npm start
