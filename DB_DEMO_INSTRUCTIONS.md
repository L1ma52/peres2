## Структура базы данных

### 📋 Таблицы БД:

#### 1. **users** - Пользователи
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT PK | Уникальный ID |
| name | VARCHAR(255) | ФИО пользователя |
| email | VARCHAR(255) | Email (уникальный) |
| password | VARCHAR(255) | Хеш пароля |
| car_brand | VARCHAR(100) | Марка автомобиля |
| car_model | VARCHAR(100) | Модель автомобиля |
| car_year | INT | Год выпуска |
| created_at | TIMESTAMP | Дата регистрации |

#### 2. **services** - Услуги
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT PK | Уникальный ID |
| service_name | VARCHAR(255) | Название услуги |
| category | ENUM | repair / tuning |
| price | DECIMAL(10,2) | Цена услуги |
| description | TEXT | Описание |

#### 3. **service_history** - История услуг
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT PK | Уникальный ID |
| user_id | INT FK | Ссылка на пользователя |
| service_id | INT FK | Ссылка на услугу |
| service_date | DATE | Дата оказания услуги |
| price_paid | DECIMAL(10,2) | Фактическая цена |
| status | ENUM | pending / completed / cancelled |
| created_at | TIMESTAMP | Дата создания записи |

#### 4. **reviews** - Отзывы
| Поле | Тип | Описание |
|------|-----|----------|
| id | INT PK | Уникальный ID |
| user_id | INT FK | Ссылка на пользователя (опционально) |
| author_name | VARCHAR(255) | Имя автора |
| rating | INT(1-5) | Оценка |
| review_text | TEXT | Текст отзыва |
| car_info | VARCHAR(255) | Информация об авто |
| created_at | TIMESTAMP | Дата создания |

### ПОСЛЕ ОФОРМЛЕНИЯ ЗАКАЗА:
```

  ✅ service_id: 8 (например, Stage 2)
  ✅ price_paid: 75000
  ✅ status: completed

История отображается в личном кабинете
```

## 📝 SQL-скрипты

Файл `database-schema.sql` содержит:

1. **Создание таблиц** - полная структура БД
2. **Наполнение данными** - тестовые данные
3. **Представления (Views)** - для удобных выборок
4. **Хранимые процедуры** - для работы с данными:
   - `register_user()` - регистрация пользователя
   - `add_service_to_history()` - добавление услуги
   - `get_user_service_history()` - получение истории
5. **Триггеры** - автоматическое логирование изменений
6. **Примеры запросов** - готовые SQL-запросы

## 🚀 Развертывание БД

### Для MySQL/MariaDB:

```bash
# Войти в MySQL
mysql -u root -p

# Выполнить скрипт
source database-schema.sql

# или
mysql -u root -p < database-schema.sql
```

### Для других СУБД:

SQL-скрипт совместим с:
- MySQL 5.7+
- MariaDB 10.2+
- PostgreSQL (с небольшими адаптациями)

## 📊 Примеры использования

### Регистрация пользователя:
```sql
CALL register_user(
    'Иванов Иван Иванович',
    'ivanov@mail.ru',
    '$2y$10$hashedpassword',
    'BMW',
    'M3',
    2020
);
```

### Добавление услуги в историю:
```sql
CALL add_service_to_history(
    1,              -- user_id
    8,              -- service_id (Stage 2)
    '2025-10-03',   -- дата
    75000.00,       -- цена
    'Отличная работа'
);
```

### Получение истории пользователя:
```sql
CALL get_user_service_history(1);
```

### Статистика по пользователю:
```sql
SELECT * FROM user_full_info WHERE user_id = 1;
```

## 💡 Рекомендации

1. **Безопасность**: Всегда храните пароли в зашифрованном виде (bcrypt, Argon2)
2. **Индексы**: Добавлены индексы на часто используемые поля
3. **Связи**: Используются FOREIGN KEY для целостности данных
4. **Каскадное удаление**: При удалении пользователя удаляется и его история
5. **Логирование**: Триггеры автоматически логируют важные изменения
