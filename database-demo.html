🗄️ Структура базы данных СТО Level

Схема PostgreSQL для управления автосервисом

Первичный ключ (PK)
Внешний ключ (FK)
Обычное поле
👤 users
Зарегистрированные пользователи системы
id
SERIAL PRIMARY KEY
full_name
VARCHAR(255) NOT NULL
email
VARCHAR(255) UNIQUE NOT NULL
password_hash
VARCHAR(255) NOT NULL
car_brand
VARCHAR(100)
car_model
VARCHAR(100)
car_year
INTEGER
created_at
TIMESTAMP DEFAULT NOW()
updated_at
TIMESTAMP DEFAULT NOW()
🔧 services
Каталог услуг автосервиса
id
SERIAL PRIMARY KEY
name
VARCHAR(255) NOT NULL
description
TEXT
price
DECIMAL(10, 2) NOT NULL
category
VARCHAR(50) NOT NULL (repair/tuning)
stage
VARCHAR(20) для тюнинга
created_at
TIMESTAMP DEFAULT NOW()
updated_at
TIMESTAMP DEFAULT NOW()
📋 service_history
История оказанных услуг
id
SERIAL PRIMARY KEY
user_id
INTEGER FK → users(id)
service_id
INTEGER FK → services(id)
service_date
TIMESTAMP DEFAULT NOW()
price
DECIMAL(10, 2) NOT NULL
status
VARCHAR(50) pending/in_progress/completed/cancelled
notes
TEXT
created_at
TIMESTAMP DEFAULT NOW()
⭐ reviews
Отзывы клиентов
id
SERIAL PRIMARY KEY
user_id
INTEGER FK → users(id) (optional)
full_name
VARCHAR(255) NOT NULL
rating
INTEGER 1-5
comment
TEXT NOT NULL
created_at
TIMESTAMP DEFAULT NOW()
✉️ contact_requests
Заявки с формы обратной связи
id
SERIAL PRIMARY KEY
name
VARCHAR(255) NOT NULL
email
VARCHAR(255) NOT NULL
phone
VARCHAR(50)
message
TEXT NOT NULL
status
VARCHAR(50) new/in_progress/completed
created_at
TIMESTAMP DEFAULT NOW()

🔗 Связи между таблицами

service_history.user_id → users.id
Связь: Каждая запись в истории услуг принадлежит пользователю (CASCADE DELETE)
service_history.service_id → services.id
Связь: Каждая запись в истории ссылается на услугу из каталога (CASCADE DELETE)
reviews.user_id → users.id
Связь: Отзыв может быть связан с пользователем (SET NULL при удалении)
